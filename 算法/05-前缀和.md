# 算法刷题笔记 · 第五类：前缀和 & 差分

> 结构同前：**直觉类比 → 核心公式 → 一维/二维 → 进阶(前缀和+哈希) → 差分 → 自检清单**。
> 前缀和是「**把重复的累加预先算好存起来**」——用 O(n) 预处理，换来每次区间求和 O(1)。

---

## 0. 一句话本质

> **前缀和 = 提前算好「从头到每个位置的累加和」，之后任意区间和都能一次相减得到。**

适用信号：
- **频繁查询**「区间 `[i,j]` 的和」
- 「和为 k 的子数组」「连续子数组之和满足某条件」
- 「区间内某种东西的个数」（把存在记为 1，区间计数就是区间和）

> 和滑动窗口的分工：滑窗适合「**全正数**、找最优区间」；前缀和（尤其配哈希）能处理「**含负数**、找和为定值的子数组」，这是滑窗做不到的。

---

## 1. 直觉类比：记录「累计里程」

开车从家出发，每经过一个路口记下**从家到这里的总里程**：

```
路口:    家   A    B    C    D
累计里程: 0    3    7    9    14
```

现在想知道「**从 B 到 D 这段开了多远**」？不用重新数，直接：

> D 的累计里程 − B 的累计里程 = 14 − 7 = **7**

这就是前缀和的全部精髓：**区间和 = 两个「累计值」相减。** 预先把累计里程记好（前缀和数组），任意两点间的距离都能一减就出来，不用每次从头加。

## 2. 一维前缀和

### 核心公式

定义 `prefix[i]` = 数组前 i 个元素之和（`nums[0] + ... + nums[i-1]`）：

```java
int[] prefix = new int[n + 1];     // 多开一位,prefix[0]=0 代表"前0个的和"
for (int i = 0; i < n; i++)
    prefix[i + 1] = prefix[i] + nums[i];
```

则**区间 `[i, j]`（含两端）的和**：

```
sum(i, j) = prefix[j + 1] - prefix[i]
```

> ⚠️ 下标对齐是最大的坑。推荐**前缀数组多开一位、`prefix[0]=0`**，这样区间 `[i,j]` 的和统一是 `prefix[j+1]-prefix[i]`，不用为「i=0」单独讨论边界。

### 例：LC 303 区域和检索（不可变）

> 多次查询数组区间 `[i,j]` 的和。预处理一次，每次查询 O(1)。

```java
class NumArray {
    int[] prefix;
    public NumArray(int[] nums) {
        prefix = new int[nums.length + 1];
        for (int i = 0; i < nums.length; i++)
            prefix[i + 1] = prefix[i] + nums[i];
    }
    public int sumRange(int i, int j) {
        return prefix[j + 1] - prefix[i];     // O(1) 查询
    }
}
```

## 3. 进阶：前缀和 + 哈希表（最高频考点）

### 例：LC 560 和为 K 的子数组

> 给数组 `nums` 和整数 `k`，求**和恰好为 k 的连续子数组的个数**（数组**可能含负数**）。
> 示例：`nums=[1,1,1], k=2` → `2`（两个 `[1,1]`）。

**思路推导**：子数组 `[i,j]` 的和 = `prefix[j+1] - prefix[i] = k`。
变形：`prefix[i] = prefix[j+1] - k`。
即：固定右端点 `j`，问题变成「**前面有多少个前缀和等于 `prefix[j+1] - k`**」。用**哈希表记录每个前缀和出现的次数**，一边遍历一边查。

```java
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> count = new HashMap<>();  // 前缀和 -> 出现次数
    count.put(0, 1);                  // ⭐ 关键:前缀和为 0 出现 1 次(空前缀)
    int prefix = 0, res = 0;
    for (int num : nums) {
        prefix += num;                            // 当前前缀和
        res += count.getOrDefault(prefix - k, 0); // 有几个旧前缀和 = prefix-k,就有几个合法子数组
        count.put(prefix, count.getOrDefault(prefix, 0) + 1);  // 记录当前前缀和
    }
    return res;
}
```

**两个关键点**：
1. **`count.put(0, 1)` 必须有**：表示「空前缀」的和是 0。否则当某个 `prefix` 本身就等于 k 时（从头开始的子数组），会漏算。这是 LC 560 的头号坑。
2. **为什么 O(n) 就够**：每个前缀和只需查一次哈希、记一次，不用两层循环枚举所有区间。

> 这道题**不能用滑动窗口**——含负数，窗口扩大和不一定增大、缩小和不一定减小，单调性没了。前缀和+哈希是处理「含负数、和为定值」的标准武器。

### ⭐ 单遍法的统一心智模型（560 / 974 / 525 都是这个骨架）

别把每道题当新东西背，它们是**同一个动作**：

> **每出现一个前缀和（作为右端点），查哈希里「能和它配对的左端点」有几个 → 有几个，就加几。** 然后把自己也存进哈希，供后续配对。

「能配对」的条件是什么，决定你**查哪个键**——这才是各题唯一的不同：

| 题 | 配对条件 | 查的键 | 存的键 |
|----|---------|--------|--------|
| LC 560 和=k | `prefix[右] - prefix[左] = k` | `prefix - k` | `prefix` |
| LC 974 被 k 整除 | `prefix[右] % k == prefix[左] % k` | `(prefix%k+k)%k` | 同左(余数) |
| LC 525 和为0最长 | `prefix[右] == prefix[左]` | `prefix` | `prefix`(只存第一次下标) |

骨架完全一样：`查 → res 累加 → 存`。换题只改「查的键」和「存值还是存下标」。

**为什么不推荐「组合数配对法」**（974 有人会想用 `C(v,2)=v*(v-1)/2` 统计同值配对）：
- 它只在条件是「**值相同**」时能用（974 余数相等、525 前缀和相等）；
- 条件是「**差为定值 k**」时（560 `prefix[左]=prefix[右]-k`，左端点值≠右端点值）**套不上组合数**，必须查 `prefix-k`。
- 单遍法两者通吃，记一个骨架就行；组合数法反而要分情况、易混淆。

> 一句话：**右端点每走一步，查「能配对的左边有几个」加进答案，再把自己存进去**。配对条件换成什么键，就解什么题。

### 3.1 LC 525 连续数组（求最长 → 哈希存「下标」）

> 二进制数组，求含**相同数量 0 和 1** 的**最长**连续子数组长度。示例：`[0,1,0]` → `2`。

**巧妙转化**：把 **0 看成 -1**，则「0 和 1 数量相等」⟺「这段子数组**和为 0**」。问题变成「求和为 0 的最长子数组」。

**和 LC 560 的关键区别**：560 求个数 → 存「前缀和出现**次数**」；525 求最长 → 存「前缀和**第一次出现的下标**」。
存第一次是因为：同一前缀和再次出现时中间这段和为 0，要最长就让左端点尽量靠前 → 只记最早位置。

```java
public int findMaxLength(int[] nums) {
    Map<Integer, Integer> firstIndex = new HashMap<>();  // 前缀和 -> 第一次出现的下标
    firstIndex.put(0, -1);            // ⭐ 前缀和0在下标-1(空前缀),让从头开始的子数组也能算
    int prefix = 0, res = 0;
    for (int i = 0; i < nums.length; i++) {
        prefix += (nums[i] == 1 ? 1 : -1);     // 0 当成 -1
        if (firstIndex.containsKey(prefix))
            res = Math.max(res, i - firstIndex.get(prefix));  // 同一前缀和,中间和为0
        else
            firstIndex.put(prefix, i);          // 只记第一次,绝不覆盖(否则左端点后移变短)
    }
    return res;
}
```

### 3.2 LC 437 路径总和 III（树上前缀和 + 回溯）

> 二叉树，求有多少条**向下路径**（父→子方向，不必从根开始/到叶子）的节点和 = `targetSum`。

**核心**：从根 DFS 到当前节点，经过的节点值就是**一个数组**，「和为 target 的路径」= 这条链上「和为 target 的子数组」→ 直接套 LC 560。维护「根到当前节点的前缀和 `curSum`」，哈希记前缀和次数，查 `count.get(curSum - target)`。

**关键不同：DFS 要回溯哈希表**。走完一棵子树回到当前节点去走另一棵时，前一棵留在哈希表里的前缀和必须**撤销**（它们不在同一条「根到节点」的链上）。这正是回溯——**进入加前缀和、离开减回去**。

```java
public int pathSum(TreeNode root, int targetSum) {
    Map<Long, Integer> count = new HashMap<>();  // 前缀和 -> 次数(long 防溢出)
    count.put(0L, 1);                            // 空前缀,对应"从根开始"的路径
    return dfs(root, 0L, targetSum, count);
}
int dfs(TreeNode node, long curSum, int target, Map<Long, Integer> count) {
    if (node == null) return 0;
    curSum += node.val;
    int res = count.getOrDefault(curSum - target, 0);       // 几个祖先前缀和=curSum-target
    count.put(curSum, count.getOrDefault(curSum, 0) + 1);   // ① 做记号:加入
    res += dfs(node.left,  curSum, target, count);          // ② 深入
    res += dfs(node.right, curSum, target, count);
    count.put(curSum, count.get(curSum) - 1);               // ③ 擦记号:离开时撤销!漏了结果偏大
    return res;
}
```

### 3.3 同款套路速记

| 题 | 转化成什么 | 哈希存什么 | 特殊点 |
|----|-----------|-----------|--------|
| LC 560 和为 K | 直接前缀和 | 前缀和 → 次数 | `put(0,1)` |
| LC 525 连续数组 | 0→-1，求和为 0 最长 | 前缀和 → **第一次下标** | 求最长，存最早位置 |
| LC 437 路径总和 III | 树上链 = 数组，套 560 | 前缀和 → 次数 | DFS 进出要**回溯**哈希 |
| LC 974 和被 K 整除 | 前缀和取模 | 前缀和 **% k 的余数** → 次数 | 负数取模要 `(r%k+k)%k` |

> 精髓一句：**固定右端点，查左边有没有合适的前缀和**。求个数存次数、求最长存下标、上了树加回溯——万变不离其宗。

### 3.4 选型提示

> 「和为定值的子数组」用滑窗还是前缀和+哈希，取决于**有没有负数**；
> 含负数时用前缀和还是 DP，取决于**求定值/个数还是求最值**。
> 这是选型判断，不属于前缀和本身的知识——完整对照表见 [README 题型识别决策表](./README.md#🧭-题型识别决策表最重要的一步)。
> 本篇只负责：确定用前缀和后，具体怎么写（560 存次数、525 存下标、437 上树加回溯）。

## 4. 二维前缀和

### 核心公式（容斥）

`prefix[i][j]` = 从左上角 `(0,0)` 到 `(i-1,j-1)` 这个矩形内所有元素之和：

```java
prefix[i][j] = prefix[i-1][j] + prefix[i][j-1] - prefix[i-1][j-1] + matrix[i-1][j-1];
//             上面那块         左面那块          减去重复的左上角     加当前格
```

查询子矩阵 `(r1,c1)` 到 `(r2,c2)` 的和（容斥，减两块再补回多减的角）：

```
sum = prefix[r2+1][c2+1] - prefix[r1][c2+1] - prefix[r2+1][c1] + prefix[r1][c1];
```

> 二维的关键是**容斥原理**：大矩形减去上、左两块，会把左上角那块**减重复**，要再加回来一次。画个图体会「减两次、补一次」。题：LC 304 二维区域和检索。

## 5. 差分（前缀和的「逆运算」）

差分专治「**对区间反复加减、最后才查询**」的场景，是前缀和的镜像。

> **差分数组 `diff`**：`diff[i] = nums[i] - nums[i-1]`。对 `diff` 求前缀和能还原出 `nums`。

**核心红利**：要给区间 `[i, j]` **整体加上 val**，不用逐个加（O(区间长度)），只需改差分数组**两个端点**（O(1)）：

```java
diff[i]   += val;     // 从 i 开始,后面都 +val
diff[j+1] -= val;     // 到 j 之后,把 +val 抵消掉
```

所有区间操作做完后，**对 diff 求一次前缀和**，就得到最终数组。

```java
// 例:LC 1109 航班预订统计 —— 多次给区间 [first,last] 加 seats
int[] diff = new int[n + 1];
for (int[] b : bookings) {
    diff[b[0] - 1] += b[2];      // 区间起点 +seats
    diff[b[1]]     -= b[2];      // 区间终点后 -seats
}
int[] res = new int[n];
res[0] = diff[0];
for (int i = 1; i < n; i++) res[i] = res[i - 1] + diff[i];  // 前缀和还原
return res;
```

> 直觉：差分只在「**变化的起点和终点**」打标记，最后前缀和把这些标记「铺开」成完整数组。
> **前缀和** = 区间频繁**查询**；**差分** = 区间频繁**修改**。一对镜像。

## 6. 自检清单（写完过一遍）

- [ ] 前缀数组**多开一位、`prefix[0]=0`** 了吗？（避免 i=0 的边界讨论）
- [ ] 区间和公式 `prefix[j+1] - prefix[i]`，下标有没有错位？
- [ ] LC 560 类：哈希表**初始化 `put(0,1)`** 了吗？（漏了会少算从头开始的子数组）
- [ ] 含**负数**的「和为定值」→ 用前缀和+哈希，别用滑窗（无单调性）。
- [ ] 二维：容斥的「**减两块、补一个角**」写对了吗？
- [ ] 差分：区间 `[i,j]` 加 val → `diff[i]+=val, diff[j+1]-=val`，终点是 `j+1` 别写成 `j`。

## 7. 一句话总结

> 前缀和 = **预存累计值，区间和靠一次相减**（记累计里程，两点一减得距离）。
> 高频杀招是**前缀和 + 哈希**：把「找和为 k 的子数组」转成「找前面有几个前缀和 = 当前 − k」。
> 它的镜像是**差分**：区间频繁修改时只标记端点，最后前缀和一次性铺开。
