## LC 128 最长连续序列（字节/腾讯高频）

> 未排序数组,求最长连续序列(如 `[100,4,200,1,3,2]` → `[1,2,3,4]` 长度 4)。要求 **O(n)**。

### 为什么不能用常规方案

- 排序后扫一遍:O(n log n),**不满足 O(n)**。
- DP/滑窗:数组无序、不连续,套不上。
- 这题的 O(n) 约束**逼出了一个哈希技巧**。

### 技巧核心:HashSet + 只从「序列起点」数

把所有数扔进 `Set`。对每个 `num`:

- 若 `num - 1` **在** Set 里 → `num` 不是起点,**跳过**(它会被自己所在序列的起点覆盖)。
- 若 `num - 1` **不在** Set 里 → `num` 是起点,往上数 `num+1, num+2, ...` 看多远。

**为什么跳过非起点**:连续序列 `[1,2,3,4]` 里只有 `1` 是起点(`0` 不在)。从 `2` 数是浪费——`2,3,4` 已经被从 `1` 数时覆盖了。

```java
public int longestConsecutive(int[] nums) {
    Set<Integer> set = new HashSet<>();
    for (int num : nums) set.add(num);

    int longest = 0;
    for (int num : nums) {
        if (!set.contains(num - 1)) {        // ⭐ 只从起点数
            int len = 1;
            while (set.contains(num + len)) len++;
            longest = Math.max(longest, len);
        }
    }
    return longest;
}
```

### 为什么 O(n)

`while` 看着像嵌套循环,但**每个数最多被「往上数」命中一次**:只有起点触发 while,而 while 数过的数(2,3,4)自己永不当起点(它们的 `num-1` 在 Set 里,被跳过)。所有 while 总步数 ≤ n。

### 反直觉在哪

直觉是「每个数都该数一数」,这题的洞察是「**绝大多数数不用数,只数起点**」。用 `num-1` 是否存在来识别起点——没见过这招很难现场推。

> 一句话记:**全扔 Set,只对 `num-1` 不在 Set 的起点往上数,总步数 O(n)。**
