# 算法刷题笔记 · 第三类：双指针

> 结构同前：**直觉类比 → 三大流派 → 模板 → 实战 → 易错点 → 自检清单**。
> 双指针和你已掌握的[滑动窗口](./01-滑动窗口.md)是亲戚——**滑窗就是「同向双指针」的一种**。这篇把整个家族梳理清楚。

---

## 0. 一句话本质

> **用两个指针代替一层嵌套循环，把 O(n²) 的暴力枚举降到 O(n)。**

核心红利：暴力解法里「两层 for 枚举所有配对」的活，很多时候能靠「两个指针有策略地移动、各自只走一遍」来完成。

适用信号：
- **有序**数组里找「两数之和 / 配对 / 区间」
- 链表找「中点 / 环 / 倒数第 k 个」
- 「原地」修改数组（去重、移动零、删元素）
- 判断回文

---

## 1. 直觉类比：两个人从两头往中间找

想象一条**从矮到高排好队**的队伍，你要找「两个人身高加起来正好 180」。

- **笨办法**：每个人和其他所有人都比一遍（O(n²)）。
- **双指针**：一个人站**最矮端**（left），一个人站**最高端**（right），看两人之和：
  - 和**太大** → 最高的那个太高了，`right` 往矮的方向挪一步
  - 和**太小** → 最矮的那个太矮了，`left` 往高的方向挪一步
  - 正好 → 找到了

每挪一步就排除一批不可能的组合，两人最多各走一遍 → **O(n)**。

> 双指针能成立的前提，往往是**有序**（或链表的结构性）——有序才能根据「大了/小了」决定指针往哪挪。

---

## 2. 三大流派（认准题型，套对模板）

| 流派 | 两指针位置 | 移动方式 | 典型题 |
|------|-----------|---------|--------|
| **对撞指针** | 一头一尾 | 相向而行，直到相遇 | 两数之和 II、三数之和、盛水容器、回文判断 |
| **快慢指针** | 同一起点 | 一快一慢，速度差 | 链表找环/中点/倒数第 k、原地去重 |
| **同向指针**（=滑窗） | 同一起点 | 都向右，维护区间 | 见[滑动窗口篇](./01-滑动窗口.md) |

> 这篇重点讲**对撞**和**快慢**，同向的看滑窗篇。

## 3. 流派一：对撞指针（一头一尾，相向而行）

### 模板

```java
int left = 0, right = n - 1;
while (left < right) {
    // 根据 left、right 处的值做判断
    if (/* 满足条件 */) {
        // 处理答案
    } else if (/* 需要更大的值 */) {
        left++;     // 左边右移，值变大（数组有序时）
    } else {
        right--;    // 右边左移，值变小
    }
}
```

### 例：LC 167 两数之和 II（输入有序）

> **LC 167**：给一个**升序排列**的数组 `numbers` 和目标值 `target`，找出两个数使它们相加等于 `target`，返回这两个数的**下标**（从 1 开始，下标小的在前）。题目保证有唯一解，且不能重复使用同一元素。
> 示例：`numbers=[2,7,11,15], target=9` → 返回 `[1,2]`（`2+7=9`）。

```java
public int[] twoSum(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left < right) {
        int sum = nums[left] + nums[right];
        if (sum == target) return new int[]{left + 1, right + 1};
        else if (sum < target) left++;    // 和小了，左指针右移变大
        else right--;                     // 和大了，右指针左移变小
    }
    return new int[]{-1, -1};
}
```

> 和暴力 O(n²) 比，这里利用**有序**：和小了只可能靠右移 left 补救，和大了只可能靠左移 right，**绝不回头**，各走一遍。

### 例：LC 11 盛最多水的容器

> **LC 11**：给一个数组 `height`，每个值代表一根竖线的高度（第 i 根在坐标 i 处）。任选两根线和 x 轴围成一个容器，问能盛多少水（面积最大是多少）。
> 示例：`height=[1,8,6,2,5,4,8,3,7]` → 输出 `49`（选高度 8 和 7 两根，底宽 7，面积 `min(8,7)×7=49`）。

两条线围水，面积 = `min(高度) × 底宽`。一头一尾开始，**移动较矮的那根**（因为面积受短板限制，移动高的只会更糟）：

```java
public int maxArea(int[] height) {
    int left = 0, right = height.length - 1, res = 0;
    while (left < right) {
        res = Math.max(res, Math.min(height[left], height[right]) * (right - left));
        if (height[left] < height[right]) left++;   // 移动短板才有可能变大
        else right--;
    }
    return res;
}
```

> 易错点：移动哪根？**永远移短的**。移长的，底宽变小、短板还在，面积必然不增——白移。

### LC 15 三数之和（对撞指针 + 固定一个数，去重是命门）

**核心降维**：`a+b+c=0` ⟺ `b+c = -a`。**固定一个数 `a`，剩下退化成「两数之和」**，用对撞指针找 `-a`。O(n³) → **O(n²)**。

**为什么先排序**：① 有序才能用对撞指针（按「大了/小了」移指针）；② 重复数变相邻，去重只需跳过相邻相同值。

**三处去重**（最容易错）：

| 位置 | 何时跳 | 为什么 |
|------|--------|--------|
| 固定数 `i` | `i>0 && nums[i]==nums[i-1]` | 同一个 a 只固定一次 |
| 左指针 `left` | 找到解后跳过 `nums[left]==nums[left+1]` | 同一个 b 配出相同三元组 |
| 右指针 `right` | 找到解后跳过 `nums[right]==nums[right-1]` | 同一个 c 配出相同三元组 |

> ⚠️ `i` 的去重和**前一个**（`i-1`）比，**不能**和后一个（`i+1`）比——否则会漏掉 `[-1,-1,2]` 这种第一个数本身就重复的合法解。

**直观理解：不去重会重复成什么样**（大前提：排序后相同的值都挨在一起）

*去重②③（左右指针）*——固定 `a=-2`、找凑成 `2` 的两数，剩余 `[0,0,0,2,2]`：
第一次 `left=0,right=2` 配出 `[-2,0,2]` ✅；若只挪一步，`left` 又指到 `0`、`right` 又指到 `2`，
再配出一个**一模一样**的 `[-2,0,2]`（只是用了不同位置的 0 和 2）。
👉 所以找到解后要 while **跳光挨着的相同值**，再换下一对不同的值。

*去重①（固定数）*——排序后 `[-1,-1,0,1,2]`：
`i=0` 固定第一个 `-1`，能凑出 `[-1,0,1]`、`[-1,-1,2]`；
`i=1` 固定第二个 `-1`，又会凑出 `[-1,0,1]`……**和 i=0 完全重复**（两个 -1 值一样）。
👉 所以 `i=1` 发现 `nums[i]==nums[i-1]` 就 `continue`，同样的固定数只当一次主角。

*为什么 i 和 `i-1` 比而非 `i+1`*：`[-1,-1,2]` 这个解**需要两个 -1**。
和 `i-1` 比：第一个 -1（i=0）正常去找，能配上第二个 -1 ✅；第二个 -1（i=1）才跳过。
和 `i+1` 比：i=0 一看后面也是 -1 就直接跳过，`[-1,-1,2]` 反而被**漏掉** ❌。

> 三处本质是同一句话：**排序后相同值挨在一起，只用其中「第一个」，后面挨着的相同值全跳过**——只是分别作用在「固定数 / 左指针 / 右指针」三个角色上。

```java
public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);                          // ① 必须先排序
    List<List<Integer>> res = new ArrayList<>();
    for (int i = 0; i < nums.length - 2; i++) {
        if (nums[i] > 0) break;                 // 剪枝:后面全正数,和不可能为 0
        if (i > 0 && nums[i] == nums[i - 1]) continue;  // 去重①:和 i-1 比
        int left = i + 1, right = nums.length - 1, target = -nums[i];
        while (left < right) {
            int sum = nums[left] + nums[right];
            if (sum == target) {
                res.add(Arrays.asList(nums[i], nums[left], nums[right]));
                while (left < right && nums[left] == nums[left + 1]) left++;   // 去重②
                while (left < right && nums[right] == nums[right - 1]) right--; // 去重③
                left++; right--;                // 跳完重复再各挪一步换新的一对
            } else if (sum < target) left++;    // 和小了,左移变大
            else right--;                       // 和大了,右移变小
        }
    }
    return res;
}
```

**易错点**：① i 去重和 `i-1` 比；② 找到解后要先 while 跳光重复再 `left++/right--`；③ 跳重复时带 `left<right` 防越界；④ `left` 起点是 `i+1`（在 i 右边找）。

## 4. 流派二：快慢指针（一快一慢，速度差）

### 前置：什么是链表

数组是连续格子，靠下标随机访问；**链表是一串珠子，每个节点装「自己的值 + 指向下一个节点的线」**，最后指向 `null` 表示结束。

```
[1|·]→[2|·]→[3|·]→[4|·]→null
```

关键限制：**链表不能用下标随机访问**，只能从头顺着线一个个走（`node = node.next`），事先也不知道有多长。这个限制正是快慢指针的用武之地。

### 这一派解决的三类问题

| 问题 | 在问什么 | 快慢指针怎么帮上忙 |
|------|---------|------------------|
| **链表找环**（LC 141） | 尾巴的线绕回前面某节点形成圈（不指向 null），判断有没有这种环 | 快 2 步慢 1 步，有环则快的**套圈追上**慢的；无环则快的先到 null |
| **链表找中点** | 不知道长度，一趟找出最中间的节点 | 快的到头时走了 2 倍路程，慢的正好在**一半**=中点 |
| **数组原地去重**（LC 26） | 有序数组**原地**（不开新数组）去掉重复，返回新长度 | 慢指针守「已整理好的队尾」，快指针**出去探路**找新值 |

```
有环长这样:  [1]→[2]→[3]→[4]
                    ↑________|     ← [4] 绕回 [2]，顺着走会永远绕圈
```

### 4.1 链表找环（Floyd 判圈 / 龟兔赛跑）

> **LC 141 环形链表**：给链表头 `head`，判断链表中**是否有环**（尾节点的 `next` 绕回前面某节点）。有环返回 `true`，否则 `false`。
> 示例：`head=[3,2,0,-4]`，尾巴 -4 连回下标 1 的节点 → `true`；`head=[1]` 单节点 → `false`。

慢指针一次走 1 步，快指针一次走 2 步。**有环**则快慢必在环内相遇（快的迟早追上慢的）；**无环**则快指针先到 null。

```java
public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;            // 慢:1 步
        fast = fast.next.next;       // 快:2 步
        if (slow == fast) return true;   // 相遇 → 有环
    }
    return false;                    // 快指针到头 → 无环
}
```

> 类比：操场上跑步，快的人迟早会从后面**套圈追上**慢的人——只要是环形跑道。直道上则快的人先冲出终点。
> 易错点：`while` 条件必须 `fast != null && fast.next != null`，否则 `fast.next.next` 空指针。

### 4.2 链表找中点

> **LC 876 链表的中间节点**：给链表头 `head`，返回**中间节点**；若长度为偶数有两个中点，返回**第二个**。
> 示例：`[1,2,3,4,5]` → 返回 `3`；`[1,2,3,4,5,6]` → 返回 `4`（两个中点 3、4 取后者）。

快指针走到头时，慢指针正好在中点（快走的是慢的 2 倍）：

```java
ListNode slow = head, fast = head;
while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
}
// slow 即中点（偶数个节点时是偏右的中点）
```

### 4.3 数组原地操作（快慢同向）

> **LC 26 删除有序数组中的重复项**：**升序**数组，**原地**删除重复项使每个元素只出现一次，返回新长度 `k`（前 k 个为去重结果，后面不管），只能用 O(1) 额外空间。
> 示例：`[0,0,1,1,1,2,2,3,3,4]` → 返回 `5`，前 5 个变为 `[0,1,2,3,4]`。

快指针探路、慢指针记录有效位置。例：LC 26 有序数组去重

```java
public int removeDuplicates(int[] nums) {
    if (nums.length == 0) return 0;
    int slow = 0;                          // slow 之前(含)都是去重后的有效区
    for (int fast = 1; fast < nums.length; fast++) {
        if (nums[fast] != nums[slow]) {    // 发现新值
            slow++;
            nums[slow] = nums[fast];       // 放到有效区末尾
        }
    }
    return slow + 1;                       // 有效长度
}
```

> 类比：`slow` 是「已整理好的队伍尾巴」，`fast` 出去探路，遇到没见过的新人就请到队尾。
> 同款套路：LC 27 移除元素、LC 283 移动零。

## 5. 三大流派速记表

| | 对撞指针 | 快慢指针 | 同向指针（滑窗） |
|---|---------|---------|----------------|
| 起始位置 | `0` 和 `n-1` | 同起点 | 同起点 |
| 移动 | 相向，`while(left<right)` | 速度差 / 探路 | 都向右，维护区间 |
| 前提 | 通常需**有序** | 链表结构 / 原地 | 区间可增量维护 |
| 干什么 | 找配对 / 区间 | 找环·中点 / 原地改 | 求最长·最短子区间 |

## 6. 通用易错点

- [ ] **对撞**：循环条件 `left < right`（相遇即停，别用 `<=` 导致同一元素自己配自己）。
- [ ] **对撞**：搞清「值变大该动谁」——有序数组里左移 left 变大、左移 right 变小，别搞反。
- [ ] **快慢链表**：`while (fast != null && fast.next != null)`，顺序不能反（短路求值防空指针）。
- [ ] **快慢原地**：`slow` 的语义要钉死（「有效区的末尾下标」还是「下一个待写位置」），返回值随之是 `slow+1` 还是 `slow`。
- [ ] 是否需要**先排序**？对撞指针类（两/三数之和、4Sum）几乎都要先 `Arrays.sort`。
- [ ] 去重：三数之和等，i / left / right 三处都要跳过相邻重复值。

---

## 7. 一句话总结

> 双指针 = **用两个有策略移动的指针，替掉一层 for 循环**。
> 三招认准：**一头一尾找配对（对撞）、一快一慢探结构（快慢）、同向滑区间（滑窗）**；
> 难点永远是「**指针该往哪挪、什么时候挪**」——而这通常由「**有序性**」或「**结构**」决定。
