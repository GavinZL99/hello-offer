# 算法刷题笔记 · 第六类：二叉树（DFS / BFS）

> 结构同前：**直觉类比 → 遍历四件套 → 两种递归思维 → 题型实战 → BST → 自检清单**。
> 二叉树是递归的「主场」——大部分树题，本质是「**信任递归会把子树处理好**」，你只管定义「当前节点该干嘛」。

---

## 0. 一句话本质

> **二叉树问题 = 在树上做 DFS（递归，深度优先）或 BFS（队列，逐层）。**
> 难点不是写循环，而是想清楚「**每个节点要做什么、向子树要什么、向上返回什么**」。

基本结构：

```java
class TreeNode {
    int val;
    TreeNode left, right;   // 每个节点最多两个孩子
}
```

适用信号：输入是树、要遍历/搜索/求路径/求深度/找祖先，几乎都是 DFS 或 BFS。

---

## 1. 直觉类比：公司组织架构

把二叉树想成**公司的汇报关系**：一个领导（节点）最多带两个直接下属（左右孩子），层层往下。

- **DFS（深度优先）**：顺着一条汇报线一插到底，到末端（叶子）再回头换另一条。像「**沿着一个部门钻到最基层，再回来看下一个部门**」。
- **BFS（广度优先）**：一层一层来，先问所有总监、再问所有经理、再问所有员工。像「**按职级一级一级开会**」。

> 求「深度 / 路径 / 子树性质」→ 多用 **DFS**；求「层级 / 最短路 / 逐层信息」→ 多用 **BFS**。

---

## 2. 遍历四件套

### 2.1 三种 DFS 遍历（前序 / 中序 / 后序）

区别只在「**访问当前节点的时机**」——在处理左右孩子之前、之间、还是之后：

```java
void dfs(TreeNode node) {
    if (node == null) return;
    // 前序:在这里访问 → 根 左 右
    dfs(node.left);
    // 中序:在这里访问 → 左 根 右
    dfs(node.right);
    // 后序:在这里访问 → 左 右 根
}
```

| 遍历 | 顺序 | 什么时候用 |
|------|------|-----------|
| **前序** 根左右 | 先处理自己再处理孩子 | 自顶向下传递信息（如路径） |
| **中序** 左根右 | — | **BST 的中序是升序**！（高频考点） |
| **后序** 左右根 | 先处理孩子再处理自己 | 自底向上汇总（如求高度、判平衡） |

> 记忆：「前中后」指的是**根**被访问的位置。前序=根在前，中序=根在中，后序=根在后。

### 2.2 BFS 层序遍历（队列实现）

逐层处理，靠一个**队列**：每次取出当前层所有节点，把它们的孩子入队作为下一层。

```java
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    while (!queue.isEmpty()) {
        int size = queue.size();           // ⭐ 先固定本层节点数
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {   // 只处理这一层
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null)  queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        res.add(level);
    }
    return res;
}
```

> ⭐ 关键是 `int size = queue.size()`：进 for 前先把本层数量定死，否则边遍历边加孩子会把层搅混。这是 BFS 分层的命门。

## 3. 两种递归思维（树题的核心心法）

写树的递归，先想清楚你是哪种思路——这比记模板重要得多。

### 思路 A：自顶向下（前序 / 传参）

「**我把信息从上往下传给孩子**」。常用一个辅助函数带参数，把「从根到当前」的累积信息往下传，**在节点处更新一个全局答案**（访问发生在处理孩子之前 → 前序）。

> 例：判断是否存在和为 target 的路径（传剩余目标，见 4.4）、保存根到叶的所有路径（传路径列表）、求最大深度的「传 depth」写法。
> 特征：**信息从根往叶流动，答案在遍历过程中更新全局变量**，而不是靠返回值。

```java
// 最大深度的"自顶向下"写法(对比 4.1 的自底向上)
int maxDepth = 0;
void dfs(TreeNode node, int depth) {
    if (node == null) return;
    depth++;                                  // 进来先更新"我的深度"
    maxDepth = Math.max(maxDepth, depth);     // 前序位置:在当前节点更新全局答案
    dfs(node.left, depth);  dfs(node.right, depth);  // 把深度往下传
}
```

> ⚠️ 注意：**求最大深度两种思路都能写**——4.1 是自底向上（返回 `max(左,右)+1`，更简洁常用），这里是自顶向下（传 depth 更新全局）。同一道题、两种解法，不矛盾。很多树题都这样，先想清楚自己用哪种思路再下笔。

### 思路 B：自底向上（后序 / 返回值）

「**我先让左右子树各自算好，把结果返回给我，我再合并**」。先递归左右孩子拿到它们的结果，再算自己的。**这是树题最常用、最强大的思路。**

```java
int maxDepth(TreeNode node) {
    if (node == null) return 0;
    int left  = maxDepth(node.left);    // 信任:左子树的深度会被算好
    int right = maxDepth(node.right);   // 信任:右子树的深度会被算好
    return Math.max(left, right) + 1;   // 合并:我的深度 = 较深的孩子 + 1
}
```

> **核心心法：信任递归。** 别在脑子里展开每一层，只需假设「`maxDepth(左)` 一定能正确返回左子树深度」，然后想「**拿到左右结果后，我这个节点该怎么算**」。想通这一步，递归就写出来了。

## 4. 题型实战

### 4.1 LC 104 二叉树的最大深度（自底向上经典）

见上方 `maxDepth`。后序思路：左右深度取大，加 1。

### 4.2 LC 226 翻转二叉树

> 把每个节点的左右孩子交换。

```java
public TreeNode invertTree(TreeNode root) {
    if (root == null) return null;
    TreeNode left  = invertTree(root.left);   // 先翻转左子树
    TreeNode right = invertTree(root.right);  // 先翻转右子树
    root.left = right;                         // 交换
    root.right = left;
    return root;
}
```

### 4.3 LC 236 最近公共祖先（LCA，后序思维的巅峰）

> 给二叉树和两个节点 p、q，找它们最近的公共祖先（最深的、同时是 p 和 q 祖先的节点）。

```java
public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;   // 找到 p 或 q,上报
    TreeNode left  = lowestCommonAncestor(root.left, p, q);    // 左子树里找
    TreeNode right = lowestCommonAncestor(root.right, p, q);   // 右子树里找
    if (left != null && right != null) return root;  // p、q 分居两侧 → 当前就是 LCA
    return left != null ? left : right;              // 都在一侧 → 返回那一侧的结果
}
```

> 直觉：每个节点问左右子树「你们各自找到 p 或 q 了吗」。**如果左右各找到一个 → 我就是分岔点 = LCA**；如果都在一边 → 答案在那一边往上冒。这是「自底向上返回值」思维的极致体现。

### 4.4 LC 112 路径总和（自顶向下传参）

> 是否存在从根到叶子、和为 target 的路径。

```java
public boolean hasPathSum(TreeNode root, int target) {
    if (root == null) return false;
    if (root.left == null && root.right == null)   // 到叶子
        return target == root.val;
    int rest = target - root.val;                   // 往下传"还差多少"
    return hasPathSum(root.left, rest) || hasPathSum(root.right, rest);
}
```

> 对比：这是**自顶向下**（把剩余目标往下传）。和 4.3 的自底向上对照着体会两种思路。
> 进阶 LC 437 路径总和 III（任意段路径计数）见[前缀和篇](./05-前缀和.md)——树上前缀和 + 回溯。

## 5. 二叉搜索树（BST）

BST 是特殊的二叉树：**每个节点，左子树所有值 < 它 < 右子树所有值**。

### 两个必记性质

1. **中序遍历 = 升序序列**（最重要！「BST + 中序」组合是高频考点）。
   - 题：LC 98 验证 BST、LC 230 第 K 小、LC 538 累加树。
2. **查找 / 插入 / 删除 = O(树高)**：利用有序性，每步能砍掉一半子树（类似二分）。

```java
// LC 700 BST 中搜索值(利用有序性,不用遍历整棵树)
public TreeNode searchBST(TreeNode root, int val) {
    if (root == null || root.val == val) return root;
    return val < root.val ? searchBST(root.left, val)    // 小了往左
                          : searchBST(root.right, val);   // 大了往右
}
```

> 看到「二叉搜索树」四个字，先条件反射两件事：**能不能用中序得到升序？能不能用有序性砍半查找？**

## 6. 网格 DFS（Flood Fill）—— DFS 从树搬到网格

岛屿类题是「图论的平民版」：不用建图，**四个方向的 DFS 就是遍历**。靠现有递归功底就能拿下，面试频率比纯图论高。

### 和二叉树 DFS 的对应关系

| 二叉树 DFS | 网格 DFS |
|-----------|---------|
| 节点有 2 个孩子（左、右） | 格子有 4 个邻居（上下左右） |
| 递归出口 `node == null` | 递归出口：**越界 或 不是目标格** |
| 访问左、右子树 | 访问上、下、左、右四个格子 |

### LC 200 岛屿数量

> `'1'`陆地 `'0'`水，求岛屿（上下左右连通的陆地）数量。

**核心思想**：扫每个格子，**每发现一个没淹过的陆地 → 岛屿数 +1，然后把整座岛「淹掉」**（DFS 把连通的 `1` 全改成 `0`），避免重复计数。

```java
public int numIslands(char[][] grid) {
    if (grid == null || grid.length == 0) return 0;
    int count = 0;
    for (int i = 0; i < grid.length; i++)
        for (int j = 0; j < grid[0].length; j++)
            if (grid[i][j] == '1') {   // 发现没淹过的陆地
                count++;               // 岛屿 +1
                dfs(grid, i, j);       // 淹掉整座岛
            }
    return count;
}
void dfs(char[][] grid, int i, int j) {
    if (i < 0 || i >= grid.length || j < 0 || j >= grid[0].length || grid[i][j] != '1')
        return;                        // 越界 或 不是陆地 → 停
    grid[i][j] = '0';                  // 淹掉当前格(标记已访问,防重复/防死循环)
    dfs(grid, i - 1, j);  dfs(grid, i + 1, j);   // 上、下
    dfs(grid, i, j - 1);  dfs(grid, i, j + 1);   // 左、右
}
```

**关键点**：
- **必须「淹掉」**（标记已访问）：否则 DFS 在同一片陆地反复横跳（死循环），主循环也会重复计数同一座岛。
- **越界检查要在访问 `grid[i][j]` 之前**（短路求值），否则下标越界。
- 不许改输入时，另开 `boolean[][] visited` 标记。复杂度 `O(m×n)`，每格最多访问一次。

> 同套路变体：岛屿最大面积（LC 695，DFS 返回面积累加）、岛屿周长（LC 463）、被围绕的区域（LC 130）。BFS 也能做（队列代替递归，同样「淹岛」）。

## 7. 自检清单（写完过一遍）

- [ ] **递归出口**写了吗？`if (node == null) return ...` 几乎是每道树题的第一行。
- [ ] 想清楚是**自顶向下（传参）**还是**自底向上（返回值）**了吗？
- [ ] 自底向上：是不是先拿到 `left`、`right` 的结果，再算自己？（别在递归前就用结果）
- [ ] **BFS** 分层：进 for 前 `int size = queue.size()` 固定本层数量了吗？
- [ ] 遇到 **BST**：考虑过「中序升序」和「有序砍半」吗？
- [ ] 叶子判断 `node.left == null && node.right == null` 和「空节点」`node == null` 别混。

---

## 8. 一句话总结

> 树题的灵魂是**信任递归**：假设子树已被正确处理，只想「当前节点该做什么、向上返回什么」。
> 两种思维——**自顶向下传参（前序）、自底向上汇总（后序）**；BFS 用队列逐层、记得固定 `size`。
> 看到 BST 就想**中序升序 + 有序砍半**。
