# 算法刷题笔记 · 第七类：二分查找

> 结构同前：**直觉类比 → 基础模板 → 找边界(左/右) → 在答案上二分 → 易错点 → 自检清单**。
> 二分看着简单，**边界要命**。本篇主线：**先把「搜索区间」定义清楚，循环条件和指针移动全都由它推出来**——不背模板，靠定义。

---

## 0. 一句话本质

> **在「有序」的搜索空间里，每次砍掉一半，把 O(n) 查找降到 O(log n)。**

适用信号：
- **有序数组**里查找 / 找插入位置 / 找边界
- 「找满足条件的**最小/最大**值」——哪怕数组无序，只要**答案有单调性**（见第 4 节「在答案上二分」）
- 旋转有序数组、找峰值等变体

> 核心前提是**单调性**：能根据 mid 处的判断，确定「答案在左半还是右半」，从而扔掉一半。

---

## 1. 直觉类比：猜数字游戏

朋友心里想一个 1~100 的数，你猜，他只回答「大了 / 小了 / 对了」。

最优策略：每次猜**中间**那个（50）。

- 他说「大了」→ 答案在 1~49，下次猜 25
- 他说「小了」→ 答案在 51~100，下次猜 75

每猜一次**砍掉一半**，100 个数最多 7 次（log₂100 ≈ 7）搞定。这就是二分——**用中点把搜索范围对半砍**。

---

## 2. 基础模板：查找确定值（LC 704）

> 有序数组里找 `target`，找到返回下标，否则返回 -1。

```java
public int search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;   // 搜索区间 [left, right] 闭区间
    while (left <= right) {                   // 闭区间,left==right 时还有一个数要查
        int mid = left + (right - left) / 2;  // 防溢出,别写 (left+right)/2
        if (nums[mid] == target) return mid;
        else if (nums[mid] < target) left = mid + 1;   // 答案在右半,左边界右移
        else right = mid - 1;                           // 答案在左半,右边界左移
    }
    return -1;
}
```

**三个必记的细节**（都由「闭区间 `[left, right]`」推出）：

| 写法 | 为什么 |
|------|--------|
| `left <= right` | 区间闭，`left==right` 时区间里还有 1 个数，得查 |
| `left = mid + 1` / `right = mid - 1` | mid 已查过、不等，从区间里**去掉它**，所以 ±1（否则死循环） |
| `mid = left + (right-left)/2` | `left+right` 可能整数溢出，这样写等价但安全 |

> ⭐ **本篇心法**：先声明「我的搜索区间是 `[left, right]`（两端都包含）」，那么 `while` 条件、指针怎么移、要不要 ±1，**全都是这句定义的推论**，不用死记。

## 3. 找边界：最容易写错的部分

「找 target」简单，但「找**第一个** ≥ target 的位置」「找 target **第一次/最后一次**出现」才是高频考点。关键变化：**找到也不停，继续往一侧逼**。

### 3.1 找左边界（第一个 == target 的下标）

```java
int leftBound(int[] nums, int target) {
    int left = 0, right = nums.length - 1, res = -1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) {
            res = mid;            // 记下来,但不返回
            right = mid - 1;      // ⭐ 继续往左找更靠前的
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return res;
}
```

### 3.2 找右边界（最后一个 == target 的下标）

和左边界**镜像**：找到后往右逼。

```java
if (nums[mid] == target) {
    res = mid;
    left = mid + 1;       // ⭐ 继续往右找更靠后的
}
```

> 记法：**找左边界 → 命中后 `right=mid-1`（往左挤）；找右边界 → 命中后 `left=mid+1`（往右挤）。**
> 用 `res` 变量记录最近一次命中，是最不容易错的写法（比纯靠 left/right 收敛后判断更稳）。

### 3.3 应用：LC 34 在排序数组中查找元素的第一个和最后一个位置

直接调用上面两个函数，返回 `[leftBound, rightBound]`。找不到返回 `[-1,-1]`。

## 4. 进阶：在「答案」上二分（最值钱的技巧）

很多题的数组本身**不**有序，但「**答案具有单调性**」——这时对**答案的取值范围**做二分。很多人不会，但一旦会了能解一大类「最小化最大值 / 最大化最小值」的难题。

### 套路三步

1. **确定答案的范围** `[lo, hi]`（答案最小可能、最大可能是多少）。
2. **写一个判定函数** `check(x)`：假设答案是 x，能不能满足条件？（返回布尔）
3. **二分这个范围**：利用 `check` 的单调性（x 越大越容易/越难满足），逼出临界值。

### 例：LC 875 爱吃香蕉的珂珂

> n 堆香蕉，第 i 堆 `piles[i]` 根。珂珂每小时吃一堆里的 k 根（吃不完整堆也只算这一堆、剩的下小时再吃）。要在 `h` 小时内吃完，求**最小的吃速 k**。

**思路**：吃速 k 越大，总耗时越短——**单调！** 对 k 二分：

```java
public int minEatingSpeed(int[] piles, int h) {
    int lo = 1, hi = 0;                      // k 最小 1,最大 = 最大那堆
    for (int p : piles) hi = Math.max(hi, p);

    while (lo < hi) {                        // 找"最小的可行 k",用 [lo,hi) 左闭右开
        int mid = lo + (hi - lo) / 2;
        if (canFinish(piles, mid, h)) hi = mid;   // mid 可行,答案可能更小,往左缩(保留mid)
        else lo = mid + 1;                         // mid 太慢,必须更大
    }
    return lo;                               // lo==hi,即最小可行吃速
}

boolean canFinish(int[] piles, int k, int h) {   // 吃速 k 能否在 h 小时吃完
    int hours = 0;
    for (int p : piles) hours += (p + k - 1) / k;  // 向上取整: ceil(p/k)
    return hours <= h;
}
```

> 思维转变：不是「在数组里找数」，而是「**在答案范围 1~max 里，二分找那个临界吃速**」。
> 同类题：LC 1011 在 D 天内送达包裹（二分运力）、LC 410 分割数组的最大值（二分那个「最大值」）、LC 1283 等。
> 识别信号：**「最小化最大值」「最大化最小值」「在 X 限制内求最优 Y」**，且「Y 越大越容易/越难满足」。

## 5. 易错点（二分的坑几乎全在边界）

- [ ] **`mid` 溢出**：用 `left + (right-left)/2`，别用 `(left+right)/2`。
- [ ] **区间定义要自洽**：用闭区间 `[l,r]` 就配 `while(l<=r)` + `l=mid+1/r=mid-1`；用左闭右开 `[l,r)` 就配 `while(l<r)` + `r=mid`。**两套别混用**，混用必死循环或漏判。
- [ ] **死循环**：当 `left = mid`（不 +1）时，若 `mid` 算的是下取整，`left` 可能永远等于 `mid` → 卡住。要么 `left=mid+1`，要么把 mid 改成上取整。
- [ ] **找边界**：命中后是往左挤还是往右挤，对应找左/右边界，别搞反。
- [ ] **在答案上二分**：`check` 函数的单调方向要想清楚（x 增大时 check 由 false→true 还是 true→false），决定 `lo=mid+1` 还是 `hi=mid`。

> 调试技巧：**拿长度为 1、2 的数组手动跑一遍**——二分的边界 bug 几乎都在最小规模暴露。

---

## 6. 两套主流写法，选一套用熟

| | 闭区间 `[l, r]` | 左闭右开 `[l, r)` |
|---|---|---|
| 初始 | `r = n - 1` | `r = n` |
| 循环 | `while (l <= r)` | `while (l < r)` |
| 收缩 | `l=mid+1` / `r=mid-1` | `l=mid+1` / `r=mid` |
| 适合 | 找确定值、找边界(配 res) | 找「第一个满足条件的位置」、在答案上二分 |

> 别两套换着写——**挑一套，把它的循环条件和收缩方式焊死在肌肉记忆里**。本篇基础用闭区间，答案二分用左闭右开，都是常见选择。

---

## 7. 一句话总结

> 二分 = **靠单调性，每次砍掉一半**。难点全在边界，破解之道是**先定义清楚搜索区间**，循环条件和指针移动都是它的推论。
> 三个层次：**找确定值（基础）→ 找左/右边界（命中不停继续挤）→ 在答案范围上二分（最值钱，认准「最小化最大值」类）**。
