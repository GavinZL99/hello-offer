# 算法刷题笔记 · 第二类：动态规划（DP）

> 结构同滑动窗口篇：**直觉类比 → 灵魂概念 → 五步法 → 一维实战 → 二维专题 → 自检清单**。
> DP 的难点从来不是「记笔记」这个动作，而是**想清楚「笔记本每一格记什么」和「这一格怎么由前面推出来」**。

---

## 0. 一句话本质

> **动态规划 = 递归（把大问题拆成小问题） + 记笔记（把小问题答案存下来，不重复算）。**

适用信号（看到这些就该想到 DP）：
- 求「**方案数**」（有多少种走法 / 组合）
- 求「**最值**」（最长 / 最短 / 最大和 / 最小代价）
- 问题能拆成「**和原问题同结构的子问题**」，且子问题会**重复出现**
- 「能否凑出 / 是否可行」（布尔型 DP）

> ⚠️ DP vs 贪心：DP 会保留所有子问题的解综合比较；贪心每步只做局部最优。
> 拿不准时先想 DP（更通用，不容易错）。

---

## 1. 直觉类比：爬楼梯时「记笔记」

算「爬到第 10 阶有几种走法」（每次走 1 或 2 阶）。

- **笨办法（暴力递归）**：第 10 阶拆成第 9 阶 + 第 8 阶，第 9 阶又拆 8、7…… **第 7 阶被反复重算无数次**。像每次有人问「3+4」你都重新掰手指。
- **聪明办法（DP）**：准备一个**笔记本**（数组 `dp`），算出一个记一个，下次直接查，绝不重算。

所有 DP 题做的都是这件事：**把重复的子问题只算一次，存起来复用。**

---

## 2. 两个灵魂概念

### ① 状态 `dp[i]` —— 笔记本每一格的含义（最重要的一步）

> 例：`dp[i]` = 爬到第 i 阶的方法数

定义清楚题就解了一半；定义错了后面全错。**先把 `dp[i]` 用一句话说清楚，再写代码。**

### ② 状态转移方程 —— 这一格怎么由前面推出

> 例：`dp[i] = dp[i-1] + dp[i-2]`

**想转移方程的万能问法**：

> **盯着 `dp[i]`，问「最后一步有哪几种选择」。** 每种选择对应一个前面的子问题。

- 爬楼梯最后一步：跨 1 步（从 i-1 来）或跨 2 步（从 i-2 来）→ 相加
- 打家劫舍最后一步：第 i 间偷（接 i-2）或不偷（接 i-1）→ 取 max

---

## 3. 解题五步法（套路模板）

| 步骤 | 问自己 | 爬楼梯的答案 |
|------|--------|-------------|
| ① **定义状态** | `dp[i]` 代表什么？ | 爬到第 i 阶的方法数 |
| ② **转移方程** | `dp[i]` 怎么由前面推出？ | `dp[i] = dp[i-1] + dp[i-2]` |
| ③ **初始化** | 最小子问题（边界）是多少？ | `dp[1]=1, dp[2]=2` |
| ④ **遍历顺序** | 从前往后还是从后往前？ | 从小到大（先有 i-1 才能算 i） |
| ⑤ **返回什么** | 答案在哪一格？ | `dp[n]` |

```java
public int climbStairs(int n) {
    if (n <= 2) return n;
    int[] dp = new int[n + 1];
    dp[1] = 1; dp[2] = 2;                       // ③ 初始化
    for (int i = 3; i <= n; i++)                // ④ 从小到大
        dp[i] = dp[i - 1] + dp[i - 2];          // ② 转移
    return dp[n];                               // ⑤ 返回
}
```

### ⭐ 计数 vs 最值：DP 的分水岭

| 问题类型 | 多个来源怎么合并 |
|----------|-----------------|
| 「有多少种」方案数 | **相加**（所有来源都算上） |
| 「最大/最小/最优」最值 | **max / min**（只选最优的一个来源） |

> 看到「有多少种」想加法；看到「最大/最小」想 max/min。这条贯穿所有 DP。

## 4. 一维实战

### 4.1 LC 198 打家劫舍

> 不能偷相邻房子，求最多偷多少钱。

```java
public int rob(int[] nums) {
    if (nums.length == 0) return 0;
    if (nums.length == 1) return nums[0];           // 挡边界
    int[] dp = new int[nums.length];
    dp[0] = nums[0];
    dp[1] = Math.max(nums[0], nums[1]);              // 严格按定义:挑大的
    for (int i = 2; i < nums.length; i++) {
        dp[i] = Math.max(dp[i - 1], dp[i - 2] + nums[i]);  // 偷 or 不偷
    }
    return dp[nums.length - 1];
}
```

转移方程「偷(接 dp[i-2]+nums[i]) or 不偷(接 dp[i-1])」取 max。

**踩坑**：
1. `dp[1]` 写成 `nums[1]`——按定义应是 `max(nums[0], nums[1])`。
   `[1,2,3,1]` 碰巧蒙对，但 `[2,1]` 立刻出错（输出 1，应为 2）。
   👉 **初始化必须严格符合 `dp[i]` 的定义**，`dp[i]` 是最大值，`dp[1]` 就得是 max。
2. `nums.length <= 1` 时 `dp[1]` 越界。👉 DP 几乎都要先挡掉小规模边界。

### 4.2 滚动数组优化：空间 O(n) → O(1)

转移只依赖**前面固定几格**时，用几个变量滚动即可，不必开整个数组：

```java
public int rob(int[] nums) {
    int prev2 = 0, prev1 = 0;     // dp[i-2], dp[i-1]
    for (int num : nums) {
        int cur = Math.max(prev1, prev2 + num);
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
```

> 爬楼梯、打家劫舍、斐波那契都适用。先用数组版想清楚，再压缩。

### 4.3 LC 53 最大子数组和（Kadane）—— 含负数，滑窗失效的经典

> 含负数的数组，求**和最大的连续子数组**。`[-2,1,-3,4,-1,2,1,-5,4]` → `6`（`[4,-1,2,1]`）。

**先识别类型**：「连续子数组 + 求最值」第一反应可能是滑窗——但**含负数 → 窗口和不单调 → 滑窗的「只进不退」会错误丢弃**（见 [01 篇](./01-滑动窗口.md)反例）。对撞双指针也不行（前缀和含负数不单调）。**正解是 DP。**

**① 定义状态**（「以 i 结尾」，和最长公共子串同款套路，锁住连续性）：
`dp[i]` = **以 nums[i] 结尾**的最大连续子数组和。

**② 转移——盯最后一步**：`nums[i]` 要不要接前面那段？
- 接上：`dp[i-1] + nums[i]`
- 自立门户：`nums[i]`（前面那段是负的，拖后腿，不如自己单干）

`dp[i] = Math.max(nums[i], dp[i-1] + nums[i])`

**③ 答案**：以 i 结尾的最大可在任意位置结尾 → **扫所有 dp[i] 取最大**（同子串，不是 dp[n-1]）。

```java
public int maxSubArray(int[] nums) {
    int dp = nums[0], res = nums[0];        // dp 滚动:以 i 结尾的最大和
    for (int i = 1; i < nums.length; i++) {
        dp = Math.max(nums[i], dp + nums[i]);   // 接上 or 自立门户
        res = Math.max(res, dp);                 // 随时记录全局最大
    }
    return res;
}
```

> **DP 为什么能行而滑窗不行**：滑窗「丢掉左边就回不来」，负数让丢弃不安全；
> DP 用 `max(自立, 接上)` 把两种都算了、不丢弃、不赌——负数的情况被 `max` 天然兜住。
> 这正是 [第 8 节](#8-dp-为什么成立最优子结构-vs-贪心核心认知)「DP 全枚举、不押注」的又一例证。

## 5. 二维 DP 专题（重点）

### 5.1 为什么会冒出第二个维度

1D 里 `dp[i]` 只有一个变化的量。但有些问题的状态需要**两个量一起才能确定**：

- **情形 A：问题本身是二维网格** —— 「机器人在 m×n 方格里走」，描述「在哪」必须同时给行 + 列。
  `dp[i][j]` = 走到第 i 行第 j 列的方法数。**表格位置直接对应网格位置**，最直观。
- **情形 B：同时处理两个序列** —— 「字符串 A 和 B 的最长公共子序列」，要同时盯 A 的第 i 位和 B 的第 j 位。
  `dp[i][j]` = A 前 i 个字符与 B 前 j 个字符的 LCS 长度。

> 类比：1D 是一个游标在一条线上滑；2D 是两个游标各管一摊，组合成一张表。

### 5.2 五步法（和 1D 一样，i 变成 i,j）

| 步骤 | 1D | 2D |
|------|-----|-----|
| ① 定义 | `dp[i]` | `dp[i][j]`（i、j 各代表什么要说清） |
| ② 转移 | 由 `dp[i-1]…` | 由 `dp[i-1][j]`、`dp[i][j-1]`、`dp[i-1][j-1]` |
| ③ 初始化 | 1~2 个起点 | **整个第一行 + 第一列** |
| ④ 遍历 | 一层循环 | **两层嵌套** |
| ⑤ 返回 | `dp[n]` | 通常 `dp[m-1][n-1]`（右下角） |

> ⭐ **2D 唯一真正变难的是 ③ 初始化**：第一行没有「上面」、第一列没有「左边」可依赖，
> 必须单独填好。这是 2D 最容易栽的地方。

### 5.3 双胞胎对比：62 不同路径 vs 64 最小路径和

骨架完全一样，只是「计数」换成「最值」：

| | 62 不同路径 | 64 最小路径和 |
|---|---|---|
| `dp[i][j]` | 到 (i,j) 的**路径数** | 到 (i,j) 的**最小数字和** |
| 转移 | `dp[i-1][j] + dp[i][j-1]`（**相加**） | `min(dp[i-1][j], dp[i][j-1]) + grid[i][j]`（**取 min** 再加自身） |
| 第一行/列 | 全填 `1` | **前缀累加**（一路走来没得选） |
| 类型 | 计数 → 加法 | 最值 → min |

**62 不同路径**：

```java
public int uniquePaths(int m, int n) {
    int[][] dp = new int[m][n];
    for (int i = 0; i < m; i++) dp[i][0] = 1;   // 第一列
    for (int j = 0; j < n; j++) dp[0][j] = 1;   // 第一行
    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
    return dp[m - 1][n - 1];
}
```

**64 最小路径和**：

```java
public int minPathSum(int[][] grid) {
    int m = grid.length, n = grid[0].length;
    int[][] dp = new int[m][n];
    dp[0][0] = grid[0][0];
    for (int i = 1; i < m; i++) dp[i][0] = dp[i - 1][0] + grid[i][0];  // 第一列前缀累加
    for (int j = 1; j < n; j++) dp[0][j] = dp[0][j - 1] + grid[0][j];  // 第一行前缀累加
    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[i][j] = Math.min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j];
    return dp[m - 1][n - 1];
}
```

> 「最后一步从哪来」对两题都一样（上面或左边）；区别只在**计数用加法、最值用 min**，
> 以及**最值的第一行/列要前缀累加**而非填 1。

### 5.4 二维 DP 三个高频坑

1. **数组开多大**：`new int[m][n]`，下标到 `m-1`/`n-1`，循环和返回别越界。
2. **第一行 + 第一列初始化**（命门）：单独用两个小循环先填好，再进双层主循环——思路最清晰。
3. **双层循环起点**：从 `i=1, j=1` 开始（第 0 行/列已初始化）。

## 6. 二维 DP 的另一大家族：两个序列

网格类（62/64）的 `dp[i][j]` 对应具体位置，很直观。两序列类更抽象——**`dp[i][j]` 同时盯着 A 的第 i 个、B 的第 j 个字符**。下面用五步法**推导**，而不是背结论。

### 6.1 LC 1143 最长公共子序列（LCS，可跳着选）

> A、B 两串，求最长**公共子序列**（可跳过字符、不要求连续）的长度。`"abcde"` 与 `"ace"` → `3`（"ace"）。

**一步步想（五步法）：**

**① 定义状态**：要描述「A、B 各匹配到哪了」需要两个变量，所以
`dp[i][j]` = A 的前 i 个字符 与 B 的前 j 个字符 的 LCS 长度。

**② 转移方程——盯着「最后一步」**：看 A 的第 i 个字符 `A[i-1]` 和 B 的第 j 个字符 `B[j-1]`，问「这两个末位字符能不能凑成公共子序列的最后一个字符」：

- **相等**（`A[i-1]==B[j-1]`）：太好了，这个共同字符可以接在 LCS 末尾 → 在「都去掉末位」的基础上 +1：`dp[i-1][j-1] + 1`。
- **不等**：这两个末位字符**不可能同时**是 LCS 的结尾，至少要舍弃一个。要么舍 A 的末位（看 `dp[i-1][j]`），要么舍 B 的末位（看 `dp[i][j-1]`），**取较大**——因为子序列允许跳过，舍掉一个还能带着前面的成果继续。

**③ 初始化**：`dp[0][j] = dp[i][0] = 0`（空串和任何串的 LCS 是 0）。
**④ 遍历**：i、j 都从小到大（要先有左/上/左上格）。
**⑤ 返回**：`dp[m][n]`（用满两串）。

```java
public int longestCommonSubsequence(String a, String b) {
    int m = a.length(), n = b.length();
    int[][] dp = new int[m + 1][n + 1];          // 多开一行一列,dp[0][*]=dp[*][0]=0 自动成立
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            if (a.charAt(i - 1) == b.charAt(j - 1))
                dp[i][j] = dp[i - 1][j - 1] + 1;             // 末位相等,接上
            else
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]); // 不等,舍一个取大
    return dp[m][n];
}
```

### 6.2 LC 最长公共子串（必须连续）

> 求最长**公共子串**（必须连续）的长度。`"abcde"` 与 `"abfce"` → `2`（"ab"）。

**先想：能直接套 LCS 的定义吗？** 不能。LCS 的 `dp[i][j]`（前 i、前 j 的最长）允许中间断开，但子串要求**连续**——我必须知道「这段公共部分是不是刚好以 `A[i-1]`、`B[j-1]` 结尾」，断没断。所以**状态定义本身要改**：

**① 定义状态（变了！）**：`dp[i][j]` = **以 `A[i-1]` 和 `B[j-1]` 为结尾**的最长公共子串长度。
（「以…结尾」是子串/连续类问题的常用定义套路，强迫连续性。）

**② 转移——还是看最后一步**：

- **相等**：可以把这个共同字符接到「以前一对字符结尾」的子串后面 → `dp[i-1][j-1] + 1`。
- **不等**：**连续断了**！以这俩字符结尾根本凑不出公共子串 → `dp[i][j] = 0`（归零，从头再来）。

**③ 初始化**：边界全 0。
**⑤ 返回（也变了！）**：答案是**整张表的最大值**，不是 `dp[m][n]`——因为子串可以在**任意位置**结尾，最长的那段不一定用到两串末尾。

```java
public int longestCommonSubstring(String a, String b) {
    int m = a.length(), n = b.length(), res = 0;
    int[][] dp = new int[m + 1][n + 1];
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++) {
            if (a.charAt(i - 1) == b.charAt(j - 1)) {
                dp[i][j] = dp[i - 1][j - 1] + 1;   // 相等,在对角线上 +1
                res = Math.max(res, dp[i][j]);     // 随时记录全表最大
            }
            // 不等时 dp[i][j] 保持 0(Java 默认),即"归零"
        }
    return res;
}
```

### 6.3 子序列 vs 子串：仅有的两处差别

| | 最长公共子序列（6.1） | 最长公共子串（6.2） |
|---|---|---|
| `dp[i][j]` 定义 | 前 i、前 j 的 LCS（可断开） | **以 A[i-1]、B[j-1] 结尾**的公共子串 |
| 末位**不相等**时 | `max(dp[i-1][j], dp[i][j-1])`（舍一个，**带着成果走**） | **`0`（断了，归零重来）** |
| 答案在哪 | `dp[m][n]`（右下角） | **整张表的最大值**（可在任意位置结尾） |

> 一句话记牢：**子串「遇不匹配归零、答案扫全表」，这是它和子序列仅有的两处不同。**
> 更深一层——**为什么定义不同**：子串要连续，必须用「以…结尾」锁住连续性；子序列能跳，所以用「前 i 前 j」就够。**定义决定了后面一切**，这也是为什么 DP 第一步「定义状态」最重要。

### 6.4 同家族其它题

| 题 | `dp[i][j]` 含义 | 转移核心 |
|----|----------------|----------|
| LC 72 编辑距离 | A 前 i 变成 B 前 j 的最少操作数 | 相等继承 `dp[i-1][j-1]`；否则 `1 + min(增/删/改)` |
| LC 1035 不相交的线 | 本质就是 LCS | 同 6.1 |

> 两序列类通用直觉：**盯着 A、B 末位字符是否匹配**——匹配走对角线 `dp[i-1][j-1]`，不匹配则在「跳过 A 末位」`dp[i-1][j]`、「跳过 B 末位」`dp[i][j-1]`（或归零）里做选择。

## 7. 背包问题（2D DP 最经典的家族）

### 7.1 状态与「放/不放」转移

> 物品 i 有重量 `w[i]`、价值 `v[i]`；背包容量 `W`；每件 0 或 1 次。求最大价值。
> `dp[i][j]` = **只考虑前 i 件物品、容量为 j** 时的最大价值。

两个维度的由来：`i` = 考虑到第几件物品；`j` = 当前还能装多重。少一维都说不清状态。

**最后一步只有两个选择**：

| 选择 | 含义 | 查哪一格 |
|------|------|---------|
| **不放** 第 i 件 | 它不影响结果 | `dp[i-1][j]`（同容量，上一行） |
| **放** 第 i 件（需 `j >= w[i]`） | 先腾出 `w[i]` 空间 | `dp[i-1][j-w[i]] + v[i]` |

```java
if (j >= w[i]) dp[i][j] = Math.max(dp[i-1][j], dp[i-1][j-w[i]] + v[i]);
else           dp[i][j] = dp[i-1][j];   // 装不下，只能不放
```

> **卡点的本质**：「放」会消耗容量，所以要去查「容量减掉它之后」那一格 `j - w[i]`。
> 类比：行李箱还剩 j 升，塞一件 w[i] 升的外套 → 塞之前只有 `j - w[i]` 升留给别的东西，
> 用那时的最优 + 外套价值，就是「放」的收益。

### 7.2 一维优化与「倒序」之坑

一维数组是「二维表的一行原地滚动刷新」：**外层 i 走一轮 = 表往下挪一行 = 多考虑一件物品**。
时间仍是 **O(N×W)**（格子数没变），空间 O(N×W) → **O(W)**。

```java
int[] dp = new int[W + 1];
for (int i = 0; i < w.length; i++)
    for (int j = W; j >= w[i]; j--)            // ⚠️ 0-1 背包：容量从大到小
        dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i]);
return dp[W];
```

**为什么倒序**：`dp[j]` 依赖更小容量 `dp[j-w[i]]`。
- **倒序**：算 `dp[j]` 时 `dp[j-w[i]]` 还是**上一行（没放物品 i）**的值 → 每件只用一次 ✅
- **正序**：`dp[j-w[i]]` 已被本行刷成「放过 i」→ 同一件被重复放 → 退化成完全背包 ❌

> 记法：**0-1 背包（每件一次）→ 一维倒序；完全背包（每件无限）→ 一维正序。** 只差遍历方向。

### 7.3 一维初始化（取决于问什么 —— 又是「初始值=空的时候按定义取值」）

| 问的是 | `dp[0]` | 其余 `dp[j]` | 原理（空背包时） |
|--------|---------|-------------|----------------|
| 最大价值·容量**至多** j | 0 | **0** | 啥都不装，价值 0（Java 默认即对） |
| 最大价值·**恰好**装满 j | 0 | **`Integer.MIN_VALUE`** | 只有容量 0 合法，其余凑不满（最后 `dp[W]<0` 即装不满） |
| **能否**凑出（布尔） | **`true`** | false | 和 0 天然可达（空集） |
| **方案数**（计数） | **1** | 0 | 凑 0 有 1 种（空集），是累加的种子 |

> 「至多」和「恰好」初始化完全不同，是背包最隐蔽的考点，审题要看清。
> 计数型 `dp[0]=1` 是新手最易漏的——设成 0 会导致全程算出来都是 0。

> 经典题：LC 416 分割等和子集（布尔型 + 一维倒序）、LC 322 零钱兑换（完全背包求最值）、
> LC 518 零钱兑换 II（完全背包求方案数）。

## 8. DP 为什么成立：最优子结构 vs 贪心（核心认知）

> 一个常见误区：「DP = 每步选局部最优，走到最后就是全局最优」。
> **这是错的——这句话描述的是「贪心」，而且贪心经常赌错。** 搞清这点，才算从「会写 DP」到「懂 DP」。

### 贪心会赌错

```
零钱兑换：硬币 = [1, 3, 4]，凑出 6，求最少硬币数
贪心(每步拿最大):  4 → 1 → 1  = 3 枚   ❌ 赌错了
DP  (所有都试):    3 + 3       = 2 枚   ✅
```

「每步拿最大」这个局部最优，在这题直接失败。所以**「局部最优走到底 = 全局最优」不普遍成立**，只有少数有「贪心选择性质」的题才行。

### DP 的核心是「最优子结构」+「试遍所有选择」

> **大问题的最优解 = 由若干「子问题的最优解」组合而成；每一步把所有选择对应的子问题最优解都拿来，取最优——所以不会赌错。**

- 零钱凑 6 = `min(凑(6-1), 凑(6-3), 凑(6-4)) + 1`（三个子问题全算，取 min）
- 打家劫舍 = `max(偷, 不偷)`（两种都算）
- LCS 不匹配 = `max(舍 A 末位, 舍 B 末位)`（两种都算，不押注单边）

### 一张表钉死区别

| | 贪心 | 动态规划 |
|---|------|---------|
| 每一步 | **选一个**当前最优，拍板 | **所有选择都试**，不拍板 |
| 会赌错吗 | 会（所以不通用） | 不会（全枚举） |
| 凭什么对 | 需要「贪心选择性质」 | **最优子结构** |

> 所以凡是写出 `max(选项A, 选项B)` / `min(...)` 的 DP，本质都是「**两种都算、取好的，从不押注单边**」。
> 这也解释了上一个困惑「LCS 会不会舍错」——**不会，因为根本没真的舍，是两条路都走了取 max**。

## 10. 一句话总结

> DP 就是**带笔记本的递归**：把大问题拆成同结构的小问题，算一个记一个、绝不重算。
> 难点永远只有两个——**「每格记什么」（状态）** 和 **「这格怎么由前面推出」（转移）**；
> 其余（初始化、遍历、降维）都是工程收口。
