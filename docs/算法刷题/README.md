# 算法刷题笔记

按「**类型**」而非「单题」组织。每篇笔记统一结构：
**直觉类比 → 通用模板 → 变体 → 实战示例 → 优化技巧 → 自检清单**。

刷题不是为了记住某道题的答案，而是建立「**看到题能归类、归类后能套模板、套不上能讲清为什么**」的能力。

---

## 🧭 题型识别决策表（最重要的一步）

> **判断错方向，后面再努力也是错的。** 拿到题先别写代码，按下面的信号对号入座。

### 第一步：看「输入结构」+「问什么」

| 看到这些信号 | 首选方案 | 篇目 |
|-------------|---------|------|
| **连续**子数组/子串 + 最长/最短满足条件（且全正数、有单调性） | 滑动窗口 | 01 |
| 子数组**和为定值** / 含**负数** / 频繁查区间和 | 前缀和（+哈希） | 05 |
| **有序**数组找配对、两/三数之和、盛水容器 | 双指针（对撞） | 03 |
| **链表**找环/中点/倒数第 k、数组原地去重 | 双指针（快慢） | 03 |
| **有序**数组查找 / 找边界 / 「最小化最大值」「在答案里找临界」 | 二分查找 | 07 |
| 求**方案数** / **最值**，且有重叠子问题、最优子结构 | 动态规划 | 02 |
| 要**列出所有**组合/排列/子集/路径（n 很小） | 回溯 | 04 |
| **树**结构：遍历/深度/路径/祖先；**网格**连通块（岛屿） | 二叉树 DFS/BFS | 06 |
| **第 K 大/小**、前 K 高频、合并 K 路、流式取最值 | 堆 / Top K | 08 |

### 第二步：最容易选错的「岔路口」，这样区分

| 纠结 A 还是 B | 判断依据 |
|--------------|---------|
| **滑窗 vs 前缀和+哈希**（都做子数组） | 全正数、找最优区间 → 滑窗；**含负数**、找和为定值 → 前缀和+哈希（滑窗的单调性没了）。原因：滑窗靠「左指针只进不退」，要求**窗口的量随增删单调**；负数破坏单调，丢掉的左边可能反悔，必须换前缀和/DP（详见 01 篇） |
| **DP vs 回溯** | 只要「**多少种 / 最优值**」→ DP；要「**把每种方案都列出来**」→ 回溯 |
| **DP vs 贪心** | 每步选局部最优能否保证全局最优？没把握 → DP（试遍所有）；能证明 → 贪心。**默认先想 DP** |
| **排序 vs 堆**（求第 K 大） | 只要前 K 个 → 堆 `O(n log k)`；要全部有序才用 sort |
| **二分能不能用** | 答案是否**单调**？「X 越大越容易满足」→ 能在答案上二分 |
| **前缀和+哈希 vs DP**（含负数子数组） | 求**定值/个数**（和=K）→ 前缀和+哈希；求**最值**（和最大）→ DP。含负数只排除滑窗，之后看问什么 |

### 「含负数的连续子数组」二维定位（高频易错，单列）

两个判断维度：**数组有没有负数 × 问什么**。

| 数组 | 问什么 | 解法 | 代表题 |
|------|--------|------|--------|
| 全正数 | 和≥K 的最短 / 和=K 的个数·最长 | **滑动窗口**（和单调，可夹逼） | LC 209 |
| 含负数 | 和=K 的**个数** | 前缀和+哈希（存次数） | LC 560 |
| 含负数 | 和=0 的**最长** | 前缀和+哈希（存第一次下标） | LC 525 |
| 含负数 | 和**最大**的子数组 | **DP**（Kadane） | LC 53 |

> 关键认知：**「含负数」只排除滑窗**（窗口和不单调，丢掉的左边可能靠负数拉回）；
> 之后用前缀和还是 DP，看「**求定值/个数** → 前缀和」还是「**求最值** → DP」。
> 详见各篇：[01 滑窗](./01-滑动窗口.md) · [02 DP](./02-动态规划.md) · [05 前缀和](./05-前缀和.md)

### 第三步：如果对不上号，问自己三个问题

1. **能不能把暴力解写出来？** 写出来后看「重复计算在哪」——重复子问题 → DP / 记忆化；重复枚举配对 → 双指针 / 哈希。
2. **答案有没有单调性？** 有 → 二分（尤其「在答案上二分」）。
3. **要的是「一个值」还是「所有具体方案」？** 一个值 → DP / 贪心 / 堆；所有方案 → 回溯 / DFS。

> 口诀：**先定类型，再套模板，最后抠边界。** 三步顺序别乱——很多人一上来就写代码，方向错了全白费。
